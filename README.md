# Claude Code Usage Analytics Platform

An end-to-end analytics platform for processing and visualizing telemetry data generated by Claude Code sessions.

This project demonstrates how raw telemetry logs can be transformed into actionable insights about developer behavior, AI usage patterns, and token consumption.


# Overview

The goal of this project is to build a pipeline that:

1. Ingests telemetry data from Claude Code sessions
2. Processes and structures raw event logs
3. Extracts analytics metrics and developer insights
4. Presents results in an interactive analytics dashboard

The final output is a **Streamlit dashboard** that allows stakeholders to explore AI usage patterns across developers, practices, and time.

---

# Architecture

Telemetry logs are processed through a simple analytics pipeline:

Telemetry Logs (JSONL)
↓
Data Processing (Python / Pandas)
↓
Analytics Aggregation
↓
Interactive Dashboard (Streamlit)

Project structure:

```
claude-code-analytics/

dashboard/
    app.py                # Streamlit analytics dashboard

src/
    process.py            # telemetry processing pipeline

data/
    raw/                  # raw telemetry logs
    processed/            # aggregated analytics datasets

requirements.txt
README.md
llm_usage_log.md
```

# Key Features

## Telemetry Processing

* Parses CloudWatch-style telemetry exports
* Extracts developer session data
* Normalizes event structures
* Handles missing fields and malformed logs

## Analytics Pipeline

The processing pipeline extracts key metrics such as:

* token usage
* session activity
* developer engagement
* time-based usage patterns

Aggregated datasets are generated for dashboard visualization.

## Interactive Dashboard

The Streamlit dashboard provides:

### Key Metrics

* Total Events
* Total Tokens Consumed
* Unique Developers
* Unique Sessions

### Visualizations

* Token usage by developer level
* Token usage by engineering practice
* Peak AI usage hours
* Token consumption over time

### Interactive Filters

Users can explore analytics using filters:

* date range
* developer practice
* developer level
* location

---

# Example Insights

From the telemetry analysis we can observe patterns such as:

* Certain engineering levels consume significantly more tokens
* AI usage tends to peak during core working hours
* Different practices adopt AI tools at different rates

These insights can help organizations better understand how AI coding tools are used across teams.

# Setup

Clone the repository:

```
git clone https://github.com/cikirose/ajinter.git
cd ajinter
```

Install dependencies:

```
python3 -m pip install -r requirements.txt
```

Run the processing pipeline:

```
cd src
python3 process.py
```

Start the dashboard:

```
streamlit run dashboard/app.py
```

The dashboard will be available at:

```
http://localhost:8501
```
